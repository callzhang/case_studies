# 

标准化产品研发流程，赋能业务提质增效​

目前，飞瓜智投已打通了抖音、快手等平台，成为了品牌电商直播中至关重要的运营管理和复盘工具。产品的高速迭代，也对产研团队的协作流程提出了更高的要求。​

以前，飞瓜智投的产研协作需要在多个平台切换，信息无法及时反馈，因此影响了研发进度。例如，飞瓜智投的测试同学一旦在第三方平台发现 bug ，就需要跳转到其他应用的群聊中告知研发同学，研发同学返回后台修改完成后，又要跳转回群聊回复测试同学，来来回回的沟通会无限消耗大家的精力和体力，导致管理者也无法及时把握整体研发流程的进度。​

如今，通过飞书任务，飞瓜智投一方面搭建了一套以月为单位的高效产研流程，通过归档“待开始、进行中、测试中、验收、已发布”等不同进度，明确相关负责人及事项优先级，切实提高了工作效率，有效缩短了开发周期。​

另一方面，在飞书群聊界面，测试同学可以直接粘贴任务 ID ，点击即可跳转查看任务进度，避免了反复切换平台带来的麻烦与不便，让工作变得无比丝滑。“这个设计超级方便，满怀对程序员的善意和爱。”飞瓜智投研发负责人李海忠分享道。​

​

多维表格“指哪打哪”，bug修复效率提升20%+​

在软件开发过程中，飞瓜智投对 bug 响应时间和处理速度有严格要求，因此需要一个易上手、响应快、可跟踪、利协作的 bug 跟踪管理工具。​

以前，飞瓜智投使用第三方平台的“泳道图”去管理项目流程，数据实时更新困难。例如，客户跟进还需要人工跨系统去登记其信息，因此很容易出现数据遗漏，无法形成准确的客户画像和系统的云端客户池。​

​

现在，多维表格为团队内部的信息互通、产品质量把控、项目复盘等场景提供强有力支撑。一方面通过表单可以高效收集 bug 问题，并自动同步到多维表格，明确权责，让团队所有成员都能及时掌握上下游信息，提升 bug 的修复效率。“你用了飞书以后整个的联动效率会非常非常高，就是指哪打哪的感觉。”​

其次，在周会或月会中，多维表格还可以生成不同可视化图表：个人修复 bug 总量、所投入工时、服务器报警信息等，方便团队了解员工产值，同时测试判断 bug 的根因也会沉淀在表格中，避免问题二次发生，提升客户服务质量和体验。​

​

“飞书+飞瓜”深度联动，实现非记忆式复盘​

直播时代，海量后台提供各种碎片化数据：曝光人数、转化率、销售额等，很多用户需要登陆 PC 端才能详细查看直播间的各项数据，这就导致移动端平台的用户使用率低且体验感不佳，产品整体 PV 和使用黏性很难提升。​

现在，通过自研 API 接口打通飞书，飞瓜智投的客户可以点击群聊置顶侧链接一键打开直播间，以 H5 的形式追踪直播间的数据，直播即点即看，数据即看即用，实现了工具与数据深度融合，客户使用成本降低，产品黏性自然大幅提升。同时，飞瓜智投还可以在直播中帮助 MCN 场控监测违规词，并以消息推送形式同步反馈到飞书群聊，最终生成直播报告文档，不仅帮助移动端用户化繁为简，节省了切换不同后台的时间，后期还可以帮助管理者实现最细颗粒度的非记忆式复盘。​

除此之外，飞瓜智投支持直接导入飞书成员架构，实现主播、助播、中控、投手不同岗位的自由排班，还可以在直播时记录单个主播的开播时间、 GMV 目标、不同时间段不同商品的观看人数、点击成交转化率等，方便管理者进行员工考核和流量投放，切实提高了团队管理的效率。​
