## 公司介绍

华润至今有 83 年的历程，是中国最大的央企之一，其业务多元，涵盖大消费、大健康、城市建设与运营、能源服务、科技与金融五大业务领域，涉足 26 个行业，下设 7 大战略业务单元、19 家一级利润中心，实体企业约 2000 家，在职员工 37.1万人。自 2017年起，华润便通过旗下华润数字科技，开发在线协同办公平台“润工作”，推动流程和信息的线上化与透明化。

如何统一审批待办？​

华润作为中国规模最大的央企之一，组织架构复杂，下设 11 个集团部室、7 大战略业务单元、19 个一级利润中心，每个利润中心（如华润置地、华润啤酒等）使用的系统都不一样，同时集团内部也有多套系统，如 OA 、督办管理、行政服务系统等。对于需要高层拍板的重要事项，以及需要一线人员知晓的紧急信息，上传下达动辄就是 10-20 个环节，分散在不同的业务系统里，决策人找起来耗时费力，容易漏消息。​

润工作 3.0 的上线，标志华润进入智能办公时代，基于飞书的开放能力，通过统一审批待办中心实现了一站式的即时、智能审批，极大提高业务流转的效率。统一待办将其他系统的待办事项统一接入，无需在多套系统之间切换，保证待办即时触达、跳转详情，同时保证抄送已阅信息的回传。​

如何统一沟通工具？​

华润曾经碰到的问题其实也是超大型组织的问题——不同部门用的沟通工具不同，而不同的工具是沟通天然的壁垒，不同业务板块、管理层级、单位和子公司间的沟通成本高，37.1 万名员工的信息散落在各个角落。​

用上了润工作 3.0 ，实现沟通与协作数字化。全集团 37.1 万人，在半年内统一用上润工作 3.0 办公，保证紧急的消息快速触达，重要的消息长期留存，搜索同事名字无需加好友即可发起会话，以即时消息为底座，整合了各项业务流的一站式入口，接入所有办公场景下的流程。​

如何整合业务系统？​

润工作 2.0 具备一定开放能力，但在性能与使用体验上有一定的局限，集成的应用也难与消息通知融合。因此与业务系统的集成，在跳转之上，需要做深度的整合，避免业务数据分散在各个信息孤岛中。​

用上了润工作 3.0 ，实现管理和业务数字化：​

  * 基于开放能力，润工作 3.0 接入 200+ 应用，构建华润丰富应用生态：对于企业自建应用，开发者只需少量改动即可对接到飞书，同时飞书小程序提供丰富的 UI 组件如富文本、地图，群组管理上也具备开放能力。飞书内部的主要产品模块也均提供了 API 、组件等不同维度的开放能力，同时飞书的 API 遵循统一的 API 规范，并保证各个 API 功能的完备性，比如群相关的 API ，提供了群的创建、群信息管理、成员管理、群配置管理的 API 能力，让开发者可以基于群 API 完成与业务系统的深度集成。 ​

如何沉淀知识资产？​

过往，员工大量依靠群进行工作沟通，在私人社交工具上有无数个工作群，包括大量临时或是重复的群，在这些群中流转着许多内部文件，造成重要信息权限不可控，且难以沉淀为企业内部规范化、体系化、可复用的知识资产。​

曾经，员工用线下的 Word 、Excel 、PPT 等工具单点创作，文件在上传下达的过程中衍生出多个版本，权限不可控，外泄风险高。对于大型央企而言，信息安全高于一切。​

润工作 3.0 ，将在多个沟通工具中分散的碎片化信息转为线上的数字化资产。可多人协同、插入多类素材的飞书在线文档改变了华润员工的工作方式和理念——以前的创作是单点的、个人的，而文档打破了人与人之间的壁垒，让信息高速流通，触达每一个人，管理者从此能基于充分的上下文完成决策。​

润工作 3.0 ，把工作中需要的各类文档云端化，每个人都能通过记录、分享、接收、创造、整合等过程，时刻记录自己的工作经验、想法、工作流程，把从前存在于头脑中的分散性知识，统一组织到一个具有系统性的企业知识网里，建立不间断的创新智慧循环，参与企业知识体系的建设。​
