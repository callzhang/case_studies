数据消费的即时性和便捷性挑战​

得到作为C 端教育服务平台，旗下有在线音视频课程、有声书、电子书、实体书、线下课程等多样化的内容服务，业务团队需即时了解商品销售情况和用户行为等，实现数据驱动经营策略。数据团队通过 DataLeap、Datawind、Datav等大数据平台加工数据，于罗盘报表大厅、指标体系等商业智能数据系统支持业务，但在数据消费的便捷性和即时性上仍然存在以下挑战：​

  1. 找数成本高：体系化的指标系统复杂度较高，定位到所需数据要先登录系统，找到对应指标看板，再找到对应指标模块，再选择时间、维度、筛选项才能拿到具体数据。​



  2. 需求和解决方案不匹配：在海量数据中，识别和提取有价值的信息对各类职能成为了关键挑战。比如，运营职能需要自助的多维分析，而业务负责人需要最及时的结果数据。需求的多样性需要匹配不同的解决方案。​



  3. 环境影响大：强依赖办公电脑访问，在客户前线等办公场合以外的场景看数体验不佳。​



​

罗盘 AI，支持识别自然语言意图，整合多系统调取数据“技能”​

为解决以上挑战，得到大数据团队先行调研并明确了以 飞书AI员工自然语言查数 的产品目标，开始确定技术方案。最初尝试行业热度较高的 “Text2SQL + RAG”方案，但 AI 生成 SQL 并查询数据的准确度和稳定性问题非常明显，即便上限很高，但并不适合投入当下的实际应用中。​

与飞书智能伙伴创建平台共创后，团队找到了解决数据准确性的全新方案，通过搭建在飞书的 Data AI Agent — “罗盘 AI”，实现“意图识别 + 技能路由”，仅让大模型来判断对话者的意图，降低不可控的AI创意供给（Text2SQL）浓度，将现有数据系统的查询能力整合为 AI Agent 可调取的技能，具体运作流程包括以下几个步骤：​

  1. 员工提问：在飞书中向罗盘 Al Agent 对话提出诉求。​



  2. 理解意图：大模型能力识别用数人的自然语言意图。​



  3. 执行技能：匹配对应的预设技能调用数据系统接口。在飞书智能伙伴创建平台中，各类数据系统的查询接口均能以“技能”方式被整合，包括：实时销售数据服务、直播数据服务、用户画像等。如此，Al Agent 可作为“数据集散中心”，匹配对应的预设技能调用数据系统接口。而且，未来上线的数据服务也可以登记为 Al Agent 中的“技能”，具备可扩展性。​



  4. 结果加工返回：数据结果以飞书卡片形式加工，返回给用数人的飞书窗口中渲染，最终交付飞书原生的高体验页面。​



这样，大模型不再充当“数据开发者”的角色，而是统领多个数据系统，根据对话匹配解决方案并自动化执行，提供低门槛、高效率、多终端的实时数据服务，数据消费者在每日必用的飞书中就能随时唤起这位“AI 数据咨询师”。​

​

对数位管理者深度访谈后，“罗盘AI”具有以下优点被认可：​

  1. AI时代的管理驾驶舱，实时掌握业务动态：管理者在飞书平台能一站式自助查询公司、部门和产品实时的业务经营性指标和流水情况，时刻掌握业务进展，快速决策。​



  2. 一句话精准拿数，一键查看体系化报表：对话式交互，且支持语音识别指令，驱动 AI 从海量数据中直接定位所需数据项，效率上更有优势；同时也推送相关体系化报表，一键查看更多关联信息。​
