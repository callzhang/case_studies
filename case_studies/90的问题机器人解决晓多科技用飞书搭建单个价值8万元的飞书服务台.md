# 

技术支持服务台帮助晓多科技解决90%重复性问题，一年节省8w成本​

作为一家 SaaS 公司，晓多科技为服务客户搭建了一支多达 350 余人的“前线部队”，包含销售、客户成功、客服，全方位解决客户在售前、售中、售后的问题。这支“前线部队”需要面对来自各行各业的客户全然不同的疑问，即便再优秀的员工，也难免有答不上来的时候，经常需要向研发、产品经理等后端同事寻求支援，其中有很多重复性问题。​

使用飞书“服务台”功能，晓多科技搭建了“技术支持服务台”。​

他们将销售、客户成功、客服等前线员工反复会问到的问题沉淀为知识库，并将其聚合到“技术支持服务台”中。服务台基于精准的语义识别可以自动应答常规问题，目前350余人的前线团队，90%的问题都没有依靠人，而是在技术支持服务台上自助解决。​

据晓多科技估算，技术支持服务台一年能为晓多科技提效55个“工日”（项目工作量的一种计量方式），一年能为晓多科技省下 8 万元。晓多还搭建了行政、HR、财务、IT等部门的服务台，让更多的问题能够得到自助解答。​

​

无需再登录 14 个系统，用飞书开放平台让办公 All in one​

飞书开放平台的应用 API 也被晓多科技挖掘出超高效潜能。基于培训、招聘等各类组织管理需求，晓多科技采买、自建了至少 14 个系统，每位员工都需要经常登录这些系统，光是记账号密码就能写满一整张便利贴。​

为了让这些系统能够顺利嫁接到飞书上，实现晓多内部的无缝协作，晓多的技术部门“玩转”了飞书开放平台。​

通过接入飞书开放平台的应用 API，这 14 个系统无需输入账号密码即可在飞书内顺畅打开，自动以飞书账号登录。不光是登录轻松无感，借助打通的 API ，这些系统内关键的信息和通知还能够即时推送到飞书的 IM 生态。例如监测服务器的系统，在收到服务器告警的信息之后，就会即时推送告警信息至技术部门内部的紧急群组。当有新人入职晓多，伴随着其开通的飞书账号，新人也会自动收到相关业务系统账号开通的推送。​
