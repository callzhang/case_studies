飞书云文档组件的开发成本解决业务痛点，达成多元化业务场景需求​

​

在华勤，多年的数据沉淀，积累了大量成体系的知识信息。为了让知识价值更大化，华勤急需一款符合企业发展规划的知识管理系统。​

基于该系统，希望能够支持：​

  1. 多类型附件（ Excel / Word / PPT / PDF 等）与图文的混合编辑排版；​



  2. 多类型附件在线预览；​



  3. 多用户在线协同编辑；​



现在，华勤选择融合飞书云文档组件&飞书 OpenAPI 达成业务目标。​

借助云文档组件，快速达成多类型存量文件转为云文档知识，并完美支撑在线预览功能。同时，飞书云文档组件保持了云文档完整的编辑功能，支持图文混合编辑排版和多用户在线协同编辑能力。低开发成本解决上述技术痛点，并达成华勤业务需求。​

飞书云文档组件在文档呈现交互方面与原生飞书保持一致，从用户角度而言，体验统一无差别，无需额外学习成本，让用户减少学习成本，快速上手新的知识管理平台。​

目前，Doc 2.0 云文档组件支持移动化查阅知识、预览附件，让知识更便捷触达到用户，方便用户在移动端进行知识预览。​

飞书服务台为员工提供优质及时的服务​

员工在日常工作中遇到了 IT、HR 相关的问题，如何快速获取帮助？如果完全依赖员工自行询问，不仅可能找不到准确的对接人，IT 和 HR 等职能人员也将面对沟通成本过高的挑战。​

现在，华勤基于飞书服务台知识库，利用 AI 机器人自动应答，释放大量人力，减少常规问题的重复回答。可快速给出相关问题的答案，经过机器人过滤后仍无法解决的问题，则需要用户提一个工单，转至人工进行专人跟进。​

经过一段时间人工回复的经验积累，反哺完善知识库，提高服务台知识库问题命中率，有效提高机器人的解决率，节省大量人工成本。​

七大领域服务台，一年时间通过机器人分别解决了 18 万条用户日常报障和咨询，减少了大量重复性的操作，有效提升企业运营效率。​

飞书移动中台为企业提供稳定支持​

伴随着华勤的数字化发展，各业务领域均投入了大量资源进行信息系统建设。为了提升系统的人员触达度及助力大办公平台的建设，往往存在如下诉求：​

  1. 重要信息通过飞书进行通知，用户可以第一时间给出反馈、进行处理。​



  2. 基于飞书的 API 能力，达成业务需求，如审批、鉴权、文档操作等。​



  3. 便捷的业务系统移动化建设，方便用户在手机上进行高效办公；​



伴随着飞书快速发展，各种能力层出不穷，但对于一些提供的 API 能力，也存在着历史版本被弃用，替换为新版本，从而导致大量系统需投入额外人力进行改造的问题。​

为了解决以上痛点，华勤规划了飞书移动中台项目，搭建起飞书 OpenAPI 与业务系统之间的对接服务。基于业务需求，抽象出通用性能力，将飞书 OpenAPI 进行封装以达成业务设想，节省大量重复性资源投入，与此同时，也避免 了由于 API 变更对下游应用造成的改造影响。​

基于移动中台的建设，打造企业的统一化接入标准。对于各应用，可以做到更好的监控与管理，配合限流、熔断等机制，保障整体服务器集群的高度稳定，在资源投入与价值产出间找到更好的平衡点。​

自飞书上线以来，一年多的时间内，全平台已上线 270+ 应用，零重大故障。经过多次飞书大版本升级，所有下游系统，皆不需要投入人力做新版本适配，大大节约了各类成本。​
