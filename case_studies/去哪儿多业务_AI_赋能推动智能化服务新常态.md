# 

挑战​

2024 年，去哪儿开始在内部大力推进 URS（User Requirement Search）体系，旨在系统化提升用户问题的识别与处理效率。尽管 URS 在理论定义、度量标准和数字化系统搭建方面已完成 0 到 1 的基础建设，并覆盖多个渠道，但在实际应用中仍面临较大挑战。由于各渠道系统相对独立，数据分散，问题排查过程严重依赖人工，导致分析效率低下。以国内机票业务为例，每日收集问题约 1460 个，但仅能完成约 35 个问题的有效归因，分析覆盖率不足 2%。外呼回访覆盖不足、问题定位慢、准确性差等问题，使得 URS 在业务支持中的作用受限，同时也加重了一线人员的负担，对业务反哺效果微弱。​

​

解决方案​

为打破人工依赖，提高问题分析覆盖率，去哪儿国内机票团队基于飞书平台构建起以“多维表格+工作流+AI 大模型”为核心的 URS 自动化归因流程。该方案集成 DeepSeek、qmoss 等工具，实现数据抓取、语义解析、AI 归因、数据核验与可视化展示的全流程自动化。具体而言，通过飞书工作流定时拉取待归因数据，借助 DeepSeek 自动解析用户回访内容，再通过 qmoss 与大模型匹配内网数据进行自动归因，最终将所有结果与中间过程记录在飞书多维表格中，便于后续运营校验与追踪。同时，飞书仪表盘将分析结果自动聚合并可视化呈现，帮助产运团队快速掌握问题分布与趋势，实现问题分析从“靠人力”向“靠系统”的结构性转变。​

​

成果​

URS 自动化归因方案上线后，去哪儿国内机票业务的用户问题分析率从 2%提升至 98%以上，实现了几乎全量问题的高效归因。原本一个问题需要耗时两小时才能分析完成，如今平均只需不到一分钟即可生成归因结果，整体人效日均提升达 96 人天。相比依赖人工经验与专业知识的分析方式，新流程大幅降低了对人员能力的依赖，使产品与运营可以将更多精力投入到问题整改和体验优化中。同时，借助飞书消息与仪表盘自动推送与展示分析结果，业务部门的响应速度与协同效率也显著提升，为 URS 在公司内部的推广奠定了基础。​

# 看数效率提升 2–6 倍：打通自研系统与数据协同链路

去哪儿在日常数据使用中存在更新滞后、移动端适配差、报表依赖人工推送等问题，影响了业务团队的响应效率和分析体验。技术运营中心数据平台部门推动将多维表格能力引入自研 BI 体系，通过开放 API 实现数据自动同步，搭配仪表盘、权限控制和移动端展示，显著优化了整体看数流程。业务同学无需等待排期开发，平均 1 天内即可自助配置报表，整体看数诉求周期效率提升 2–6 倍，日常报表协作也更加高效透明，预计可节省 50 人日以上的系统工时。该方案已覆盖机票、酒店等十余业务线，成为去哪儿内部重要的数据使用通道。​

# 每月节省 44 人日：多项目会议与协同流程全链路自动化
